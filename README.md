## 📖 简介

天玄宗是一个基于修仙宗门设定的 Claude Code 插件，通过 7 个专业角色的协作，为软件开发提供完整的工作流支持。

### ✨ 核心特性

- 🏯 **多 Agent 协作** - 7 个专业角色，分工明确，高效协作
- 🎨 **中文修仙主题** - 独特的文化特色，使用体验更有趣
- 🛡️ **HARD-GATE 原则** - 绝不猜测、绝不占位、绝不跳过
- 📚 **藏经阁机制** - 强制知识复用，避免重复调研
- ✅ **两阶段审查** - 先功能后质量，确保交付标准
- 🧪 **TDD 工作流** - 测试驱动开发，质量有保障
- 📝 **14 个专业技能** - 覆盖需求、开发、测试、审查全流程

## 🚀 快速开始

### 安装

详见 [安装指南](INSTALL.md)

### 初始化

```bash
# 1. 创建天玄宗目录结构
/宗门初始化

# 2. 勘查项目灵脉，生成灵脉信息
/灵脉勘查
```

`/宗门初始化` 会创建天玄宗的工作目录结构：

```
天玄宗/
├── 宗门任务榜/        # 任务管理
│   ├── 进行中/
│   ├── 已完成/
│   └── 计划/
├── 藏经阁/            # 知识库
│   ├── 功法大全/
│   └── 最佳实践/
├── 执法堂/            # 测试报告
└── 丹堂/              # 数据分析
```

`/灵脉勘查` 会分析项目并生成文档到藏经阁：
- 项目概览.md - 项目基本信息和技术栈
- 架构设计.md - 系统架构和模块划分
- 技术栈.md - 详细的技术栈清单
- 开发规范.md - 代码规范和最佳实践
- API文档.md - API 端点和接口说明
- 数据模型.md - 数据库结构和实体关系

### 第一个任务

```
用户: 我想实现一个用户登录功能

宗主: 收到！让我先了解一下需求...
      - 使用什么认证方式？JWT 还是 Session？

      3. 执法堂负责测试和质量把关
      
      开始执行...
```

## 💡 核心概念

### HARD-GATE 原则

天玄宗的所有角色都遵循 HARD-GATE 原则：

```
✅ 绝不猜测 - 不确定就问，不自作主张
✅ 绝不占位 - 不写 TODO/FIXME/placeholder
✅ 绝不跳过 - 严格按步骤执行，不走捷径
```

### 藏经阁机制

所有技术调研和最佳实践都存储在藏经阁：

1. **开始任务前** - 必须先查询藏经阁
2. **发现新知识** - 由藏经阁长老调研并存储
3. **知识复用** - 避免重复调研，提高效率

### 两阶段审查

执法堂长老采用两阶段审查法：

1. **第一阶段：功能审查** - 验证功能是否符合需求
2. **第二阶段：质量审查** - 检查代码质量、测试覆盖、文档完整性

## 🎓 Skills 系统

天玄宗拥有 14 个专业技能，覆盖完整的开发流程：

### 需求与计划
- **需求探索术** - 需求探索、设计确认（宗主）
- **任务计划编写术** - 编写详细实现计划（宗主）
- **任务执行法** - 执行任务计划（宗主）

### 开发
- **藏经阁优先法** - 强制查询藏经阁（阵堂、器堂、杂役）
- **自检心法** - 完成后自检清单（阵堂、器堂、杂役）
- **代码审查请求术** - 请求代码审查（阵堂、器堂）
- **代码审查接收法** - 接收审查反馈（阵堂、器堂）

### 测试与质量
- **测试驱动开发法** - TDD 红绿重构（执法堂）
- **两阶段审查法** - 先功能后质量（执法堂）
- **系统化调试术** - 诊断根因、打回修复（执法堂）
- **完成前验证法** - 最终验证（宗主、执法堂）

### 工具
- **Git工作流** - Git 分支管理（所有堂）
- **代码审查诀** - 代码审查清单（执法堂）
- **文档生成法** - 文档自动生成（所有堂）

详见 [Skills 完整总结](SKILLS_COMPLETE_SUMMARY.md)

## 🔄 完整工作流程

```
1. 用户提需求
   ↓
2. 宗主（需求探索术）
   - 提问了解需求
   - 提出设计方案
   - 获得用户确认
   ↓
3. 宗主（任务计划编写术）
   - 创建详细实现计划
   - 分配任务给各堂
   ↓
4. 执法堂（测试驱动开发法）
   - 编写测试用例
   - 运行测试（红）
   ↓
5. 阵堂/器堂
   - 藏经阁优先法（查询最佳实践）
   - 查看测试用例
   - 实现功能
   - 自检心法（自我检查）
   ↓
6. 执法堂
   - 运行测试（绿）
   - 两阶段审查法（功能 + 质量）
   - 代码审查诀（详细审查）
   ↓
7. 阵堂/器堂
   - 代码审查接收法（处理反馈）
   - 修复问题
   ↓
8. 执法堂
   - 完成前验证法（全面检查）
   ↓
9. 宗主
   - 完成前验证法（最终验收）
   - 归档到藏经阁
```
