# 灵脉勘查命令

## 概述

`/灵脉勘查` 命令如同修仙者勘查灵脉一般，深入分析当前项目的完整结构、技术栈、架构设计和关键代码，并将勘查所得的知识存储到藏经阁中，供后续开发参考。

## 执行流程

### 第一阶段：宗主主导项目分析

宗主会：

1. **扫描项目结构**
   - 识别项目根目录
   - 分析目录结构
   - 识别关键配置文件（package.json, pom.xml, requirements.txt 等）
   - 检测技术栈和框架

2. **分析技术架构**
   - 识别前端/后端技术栈
   - 分析项目分层结构
   - 识别核心模块和依赖关系
   - 检测设计模式和架构风格

3. **提取关键信息**
   - 项目名称和描述
   - 主要功能模块
   - API 端点和路由
   - 数据模型和数据库结构
   - 配置和环境变量

4. **识别开发规范**
   - 代码风格和格式化规则
   - 命名约定
   - 目录组织规范
   - 测试策略

### 第二阶段：藏经阁长老存储文档

藏经阁长老会：

1. **创建项目文档**
   - 在 `天玄宗/藏经阁/项目文档/` 下创建以下文档：
     - `项目概览.md` - 项目基本信息和技术栈
     - `架构设计.md` - 系统架构和模块划分
     - `技术栈.md` - 详细的技术栈清单
     - `开发规范.md` - 代码规范和最佳实践
     - `API文档.md` - API 端点和接口说明
     - `数据模型.md` - 数据库结构和实体关系

2. **更新藏经阁索引**
   - 更新 `天玄宗/藏经阁/INDEX.md`
   - 添加文档路径和关键词
   - 记录更新时间

3. **生成快速参考**
   - 创建 `快速参考.md` 包含：
     - 常用命令
     - 重要文件路径
     - 核心概念速查
     - 常见问题解答

## 配置选项

可以通过参数自定义勘查深度：

```bash
# 快速勘查（仅基本信息）
/灵脉勘查 --quick

# 深度勘查（包含代码质量评估）
/灵脉勘查 --deep

# 仅更新特定部分
/灵脉勘查 --update api
/灵脉勘查 --update database
```

## 注意事项

1. **首次运行时间较长**
   - 项目越大，分析时间越长
   - 建议在项目初始化时运行一次

2. **定期更新**
   - 项目结构变化后建议重新运行
   - 新增重要模块后更新文档

3. **隐私保护**
   - 不会上传代码到外部
   - 敏感信息会自动脱敏
   - 所有文档存储在本地

4. **与 /init-tianxuan 的区别**
   - `/init-tianxuan` 创建天玄宗目录结构
   - `/灵脉勘查` 勘查项目灵脉并生成文档
   - 建议先运行 `/init-tianxuan`，再运行 `/灵脉勘查`

## 实现原理

1. **宗主** 使用代码分析工具扫描项目
2. **宗主** 调用 `藏经阁优先法` 检查是否已有文档
3. **宗主** 生成结构化的项目信息
4. **藏经阁长老** 将信息格式化为 Markdown 文档
5. **藏经阁长老** 更新索引和关键词
6. **藏经阁长老** 生成快速参考手册
