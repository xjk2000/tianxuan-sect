## 🔄 完整工作流程

### 1. 需求阶段
```
用户 → 宗主（需求探索术）
      ↓
   设计确认
      ↓
   任务计划编写术
      ↓
   详细实现计划
```

### 2. 开发阶段
```
宗主（任务执行法）
  ↓
分配任务
  ↓
执法堂:
  1. 测试驱动开发法（编写测试用例）
  2. 运行测试（红 - 确认失败）
  3. 记录测试报告
  ↓
阵堂/器堂:
  1. 藏经阁优先法（查询最佳实践）
  2. 查看执法堂的测试用例
  3. 实现功能
  4. 自检心法（自我检查）
  5. 通知执法堂测试
  ↓
执法堂:
  1. 运行测试（绿 - 确认通过）
  2. 更新测试报告
```

### 3. 审查阶段
```
执法堂:
  1. 两阶段审查法（功能 + 质量）
  2. 代码审查诀（详细审查）
  3. 系统化调试术（发现问题）
  ↓
反馈给阵堂/器堂
  ↓
阵堂/器堂:
  代码审查接收法（修复问题）
```

### 4. 验收阶段
```
执法堂:
  完成前验证法（全面验证）
  ↓
宗主:
  完成前验证法（最终验收）
  ↓
归档到藏经阁
```

## 🚀 使用示例

### 完整开发流程

```
1. 用户提需求
   @宗主 我想实现用户管理系统

2. 宗主使用需求探索术
   - 提问了解需求
   - 提出设计方案
   - 获得用户确认

3. 宗主使用任务计划编写术
   - 创建详细实现计划
   - 保存到 天玄宗/宗门任务榜/计划/

4. 宗主使用任务执行法
   - 分配任务给阵堂和器堂

5. 阵堂长老执行任务
   - 使用藏经阁优先法查询最佳实践
   - 使用测试驱动开发法开发
   - 使用自检心法自检
   - 使用代码审查请求术请求审查

6. 执法堂长老审查
   - 使用两阶段审查法验证
   - 使用代码审查诀详细审查
   - 提供反馈

7. 阵堂长老修复
   - 使用代码审查接收法处理反馈
   - 修复问题
   - 重新请求审查

8. 执法堂最终验证
   - 使用完成前验证法全面检查

9. 宗主验收
   - 使用完成前验证法最终验收
   - 归档到藏经阁
```
