# 杂役弟子使用指南

## 角色定位

杂役弟子是天玄宗的辅助角色，负责处理宗主认为足够简单的开发任务。

## 何时使用杂役弟子

### ✅ 适合的场景

1. **简单的文件操作**
   ```
   @杂役弟子 创建 .prettierrc 配置文件
   @杂役弟子 更新 README 添加安装说明
   @杂役弟子 删除过期的临时文件
   ```

2. **简单的代码修改**
   ```
   @杂役弟子 将所有 var 改为 const
   @杂役弟子 统一代码缩进为 2 空格
   @杂役弟子 添加函数注释
   ```

3. **简单的工具函数**
   ```
   @杂役弟子 添加一个日期格式化函数
   @杂役弟子 实现一个简单的数组去重函数
   ```

4. **文档工作**
   ```
   @杂役弟子 更新 CHANGELOG
   @杂役弟子 整理 API 文档
   @杂役弟子 添加代码示例
   ```

### ❌ 不适合的场景

1. **复杂的业务逻辑**
   ```
   ❌ @杂役弟子 实现用户权限系统
   ✅ @阵堂长老 实现用户权限系统
   ```

2. **需要设计的功能**
   ```
   ❌ @杂役弟子 设计数据库结构
   ✅ @宗主 设计数据库结构
   ```

3. **测试相关**
   ```
   ❌ @杂役弟子 编写测试用例
   ✅ @执法堂长老 编写测试用例
   ```

## 任务复杂度判断

### 简单任务（杂役弟子）

- ✅ 明确定义的操作
- ✅ 有现成的模式可遵循
- ✅ 不需要设计决策
- ✅ 15 轮对话内可完成

### 中等任务（长老）

- ⚠️ 需要一定的设计
- ⚠️ 涉及多个文件
- ⚠️ 需要技术选型
- ⚠️ 需要 20-30 轮对话

### 复杂任务（宗主 + 多个长老）

- ❌ 需要架构设计
- ❌ 涉及多个模块
- ❌ 需要团队协作
- ❌ 需要多次迭代
