# 藏经阁优先 (Knowledge First)

## 技能说明

强制要求各堂长老在开始任务前，必须先查询藏经阁相关文档。

## 核心原则

**先查询再实现** - 避免重复调研，复用已有知识。

## 执行流程

### 1. 任务开始前强制查询

各堂长老在读取任务文件后，**必须**先查询藏经阁：

```
@藏经阁长老 查询以下文档:
- [相关技术栈] 最佳实践
- [第三方平台] API 文档
- [项目经验] 相关经验
```

### 2. 等待藏经阁响应

藏经阁长老会返回：

- **找到文档**: 返回文档路径和摘要
- **未找到文档**: 提示等待或手动查询

### 3. 基于文档实现

根据藏经阁返回的文档进行实现：

- ✅ 参考文档中的最佳实践
- ✅ 使用文档中的示例代码
- ✅ 遵循文档中的规范
- ✅ 避免文档中提到的陷阱

## 未找到文档时的处理

**核心原则**: 如果藏经阁未找到相关文档，**必须由藏经阁自己去查询官方文档并存储**，其他堂不得自行查询。

### 正确流程

```
阵堂长老: @藏经阁长老 查询 Express 最佳实践

藏经阁长老: 未找到相关文档，我立即查询官方文档并存储。

(藏经阁查询官方文档)
(藏经阁总结并存入 天玄宗/藏经阁/技术总结/)
(藏经阁更新 INDEX.md)

藏经阁长老: 已完成调研并存储。文档路径: 天玄宗/藏经阁/技术总结/Express-最佳实践.md

阵堂长老: 收到，我现在基于藏经阁的文档开始实现。
```

### ❌ 错误做法

```
藏经阁长老: 未找到相关文档。

阵堂长老: 好的，我自己去查询官方文档。  ❌ 错误！
```

**为什么这是错误的**：
1. ❌ 破坏了知识沉淀机制 - 查询结果没有存入藏经阁
2. ❌ 造成重复调研 - 下次其他堂还要重新查询
3. ❌ 违反职责分离 - 各堂应该专注于自己的职责

### 藏经阁的责任

当未找到文档时，藏经阁必须：

1. **立即查询** - 使用 Web 工具查询官方文档
2. **总结存储** - 将查询结果总结并存入对应目录
3. **更新索引** - 更新 INDEX.md
4. **通知调用者** - 告知文档已准备好

### 示例：藏经阁查询并存储

```bash
# 1. 查询官方文档
(使用 Web 工具查询 Express 官方文档)

# 2. 总结并存储
cat > 天玄宗/藏经阁/技术总结/Express-最佳实践.md <<EOF
# Express 最佳实践

## 路由设计
...

## 错误处理
...

## 中间件使用
...
EOF

# 3. 更新索引
echo "- Express 最佳实践 - 技术总结/Express-最佳实践.md - $(date)" >> 天玄宗/藏经阁/INDEX.md

# 4. 通知调用者
echo "✅ 已完成调研并存储"
```

## 好处

1. **避免重复调研** - 复用已有知识
2. **保持一致性** - 遵循统一规范
3. **提高质量** - 参考最佳实践
4. **加快速度** - 减少探索时间
5. **知识沉淀** - 持续积累经验

## 注意事项

1. **必须等待响应** - 不要跳过查询步骤
2. **认真阅读文档** - 不要只是形式上查询
3. **灵活应用** - 根据实际情况调整
4. **反馈改进** - 发现文档问题及时反馈
