# 任务执行法 (Executing Plans)

## 技能说明

执行由任务计划编写术创建的实现计划，逐步完成每个任务。

## 核心原则

**严格按照计划执行，不跳过步骤，不偷工减料。**

## 执行流程

### 1. 加载计划

```
宗主: 加载计划文件

路径: 天玄宗/宗门任务榜/计划/2024-03-28-用户管理系统.md

检查:
- [ ] 计划文件存在
- [ ] 计划格式正确
- [ ] 所有任务清晰
```

### 2. 任务分配与并行判断

```
宗主: 根据任务类型分配给对应的堂，并判断是否可并行

任务 1-2 (后端): @阵堂长老
任务 3 (前端): @器堂长老【可与任务 1-2 并行】
任务 4 (测试): @执法堂长老【等待任务 1-3 完成】

并行判断:
- 阵堂和器堂都基于契约文件执行 → 并行调度
- 器堂依赖阵堂真实实现 → 串行调度
```

### 3. 执行任务

每个堂长老执行分配的任务：

```
阵堂长老: 执行任务 1

步骤 1: 写失败的测试
- [ ] 创建测试文件
- [ ] 编写测试代码
- [ ] 保存文件

步骤 2: 运行测试验证失败
- [ ] 运行测试命令
- [ ] 确认失败原因正确
- [ ] 截图或记录输出

步骤 3: 实现最少代码
- [ ] 创建实现文件
- [ ] 编写最少代码
- [ ] 保存文件

步骤 4: 运行测试验证通过
- [ ] 运行测试命令
- [ ] 确认所有测试通过
- [ ] 检查输出无警告

步骤 5: 提交
- [ ] git add 相关文件
- [ ] git commit 带清晰消息
- [ ] 推送到远程（如需要）
```

### 4. 自检

每个任务完成后，执行自检：

```
阵堂长老: 任务 1 完成，执行自检

(使用自检心法)

- [ ] 代码符合规范
- [ ] 测试全部通过
- [ ] 没有 TODO/FIXME
- [ ] 没有调试代码
- [ ] 已提交代码
```

### 5. 报告宗主

```
阵堂长老: @宗主 任务 1 已完成

完成内容:
- ✅ 用户服务层实现
- ✅ 所有测试通过
- ✅ 代码已提交

提交记录:
- feat: 添加用户创建功能 (abc123)
```

### 6. 宗主审查

**串行任务**：
```
宗主: 审查阵堂的任务 1

检查:
- [ ] 代码符合设计
- [ ] 测试覆盖充分
- [ ] 提交消息清晰
- [ ] 没有遗留问题

结果: ✅ 通过

继续下一个任务。
```

**并行任务**：
```
宗主: 阵堂和器堂都已完成并行开发

检查:
- [ ] 阵堂代码符合设计
- [ ] 器堂代码符合设计
- [ ] 契约一致性（前后端接口匹配）
- [ ] 测试全部通过

结果: ✅ 全部通过

调度执法堂开始验收。
```

## 执行检查点

### 每个步骤后

- [ ] 步骤完全按计划执行
- [ ] 没有跳过任何细节
- [ ] 输出符合预期

### 每个任务后

- [ ] 所有步骤完成
- [ ] 自检通过
- [ ] 代码已提交
- [ ] 报告宗主

### 所有任务后

- [ ] 所有任务完成（包括并行任务的所有分支）
- [ ] 并行任务的契约一致性已检查
- [ ] 集成测试通过
- [ ] 文档已更新
- [ ] 准备验收

## 遇到问题时

### 问题 1: 测试没有按预期失败

```
阵堂长老: 测试应该失败但通过了

处理:
1. 检查测试代码是否正确
2. 检查是否有现有实现
3. 修正测试或删除现有代码
4. 重新运行

不要继续，必须先解决。
```

### 问题 2: 实现后测试仍然失败

```
阵堂长老: 实现了代码但测试还是失败

处理:
1. 检查错误消息
2. 检查实现是否符合测试
3. 调试找到问题
4. 修正实现
5. 重新运行

不要修改测试来适应错误的实现。
```

### 问题 3: 计划中的代码不完整

```
阵堂长老: 计划中缺少某些细节

处理:
1. 报告宗主
2. 请求补充计划
3. 等待更新后继续

不要自行猜测或补充。
```

### 问题 4: 发现设计问题

```
阵堂长老: 执行中发现设计有问题

处理:
1. 立即停止执行
2. 报告宗主
3. 说明问题和建议
4. 等待宗主决定

不要擅自修改设计。
```

## 批量执行模式

对于简单任务，可以批量执行：

```
阵堂长老: 批量执行任务 1-3

每个任务:
1. 执行所有步骤
2. 自检
3. 提交

完成后统一报告宗主。

检查点:
- 任务 1 完成 ✅
- 任务 2 完成 ✅
- 任务 3 完成 ✅

报告: @宗主 任务 1-3 已完成
```

## 与其他技能的配合

### 配合藏经阁优先法

```
阵堂长老: 开始任务前先查询藏经阁

@藏经阁长老 查询:
- Express 最佳实践
- JWT 使用方法
- bcrypt 加密方法

(等待藏经阁响应)

(基于藏经阁文档执行任务)
```

### 配合测试驱动开发法

```
阵堂长老: 执行任务时遵循 TDD

(使用测试驱动开发法)

1. 先写测试
2. 看它失败
3. 写代码使其通过
4. 重构

严格遵循红-绿-重构循环。
```

### 配合自检心法

```
阵堂长老: 任务完成后自检

(使用自检心法)

检查:
- [ ] 没有 TODO/FIXME
- [ ] 没有调试代码
- [ ] 代码符合规范
- [ ] 测试全部通过
```

## 质量保证

### 执行前

- [ ] 理解计划
- [ ] 准备环境
- [ ] 查询藏经阁

### 执行中

- [ ] 严格按步骤
- [ ] 每步都验证
- [ ] 遇到问题立即报告

### 执行后

- [ ] 自检通过
- [ ] 测试通过
- [ ] 代码提交
- [ ] 报告宗主

## 总结

任务执行法的关键：
- ✅ 严格按计划
- ✅ 不跳过步骤
- ✅ 每步都验证
- ✅ 遇到问题立即报告
- ✅ 完成后自检
- ✅ 及时报告宗主
- ✅ 可并行任务默认并行执行，不串行化
