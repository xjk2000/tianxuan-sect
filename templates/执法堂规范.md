## 两阶段测试流程

### 第一阶段：功能测试

#### 1. 单元测试验证

```bash
# 步骤：
1. 运行所有单元测试
2. 检查测试覆盖率是否达标
3. 检查是否有失败的测试用例
4. 记录测试结果
```

**通过标准**:
- ✅ 所有单元测试通过
- ✅ 代码覆盖率达到要求
- ✅ 无测试用例失败

#### 2. 集成测试验证

```bash
# 步骤：
1. 启动测试环境（数据库、Redis 等）
2. 运行集成测试
3. 验证 API 接口功能
4. 验证数据库操作
5. 记录测试结果
```

**通过标准**:
- ✅ 所有集成测试通过
- ✅ API 接口响应正确
- ✅ 数据持久化正常

#### 3. E2E 测试验证

```bash
# 步骤：
1. 启动完整应用（前端 + 后端）
2. 运行 E2E 测试脚本
3. 验证用户操作流程
4. 检查页面交互
5. 记录测试结果
```

**通过标准**:
- ✅ 所有 E2E 测试通过
- ✅ 用户流程完整可用
- ✅ 页面交互正常
