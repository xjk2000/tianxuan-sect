# 宗门任务推演

## 概述

`/宗门任务推演` 命令用于对当前开发成果进行**测试驱动验证**。执法堂长老会：
1. **编写测试用例**（TDD 方式）
2. **开发测试脚本**
3. **启动项目并执行测试**
4. **生成测试报告**

## 🎯 核心理念

**测试驱动开发 (TDD)**：
- ✅ 先写测试，再验证实现
- ✅ 测试用例即需求规范
- ✅ 自动化测试保证质量

## 🚨 重要提醒给执法堂长老

**TDD 工作流程**：

1. **Phase 1: 编写测试用例**
   - 基于任务需求编写测试用例
   - 覆盖正常流程、边界情况、异常处理
   - 测试用例即验收标准

2. **Phase 2: 开发测试脚本**
   - 后端测试：JUnit/TestNG + MockMvc
   - 前端测试：Jest/Vitest + Testing Library
   - 集成测试：端到端测试脚本

3. **Phase 3: 启动项目并测试**
   - 启动后端服务
   - 启动前端服务
   - 执行测试套件

4. **Phase 4: 生成测试报告**
   - 测试覆盖率报告
   - 测试结果汇总
   - 问题清单和修复建议

### 第二阶段：编写测试用例（执法堂长老）

**TDD 原则：先写测试，明确预期**

**后端测试用例示例**：

```java
// src/test/java/com/example/YCloudIntegrationTest.java

@SpringBootTest
@AutoConfigureMockMvc
class YCloudIntegrationTest {
    
    @Autowired
    private MockMvc mockMvc;
    
    @Test
    @DisplayName("测试发送 WhatsApp 消息 - 正常流程")
    void testSendWhatsAppMessage_Success() throws Exception {
        // Given: 准备测试数据
        String requestBody = """
            {
                "to": "+1234567890",
                "content": "Hello from YCloud"
            }
            """;
        
        // When: 调用接口
        mockMvc.perform(post("/api/ycloud/send")
                .contentType(MediaType.APPLICATION_JSON)
                .content(requestBody))
                // Then: 验证结果
                .andExpect(status().isOk())
                .andExpect(jsonPath("$.code").value(200))
                .andExpect(jsonPath("$.data.messageId").exists());
    }
    
    @Test
    @DisplayName("测试发送 WhatsApp 消息 - 参数校验失败")
    void testSendWhatsAppMessage_InvalidParams() throws Exception {
        // Given: 无效的手机号
        String requestBody = """
            {
                "to": "invalid",
                "content": "Hello"
            }
            """;
        
        // When & Then
        mockMvc.perform(post("/api/ycloud/send")
                .contentType(MediaType.APPLICATION_JSON)
                .content(requestBody))
                .andExpect(status().isBadRequest())
                .andExpect(jsonPath("$.code").value(400));
    }
    
    @Test
    @DisplayName("测试接收 Webhook 回调")
    void testReceiveWebhook() throws Exception {
        // Given: YCloud webhook 数据
        String webhookBody = """
            {
                "event": "message.delivered",
                "messageId": "msg_123",
                "timestamp": "2026-03-30T10:00:00Z"
            }
            """;
        
        // When & Then
        mockMvc.perform(post("/api/ycloud/webhook")
                .contentType(MediaType.APPLICATION_JSON)
                .content(webhookBody))
                .andExpect(status().isOk());
    }
}
```

**前端测试用例示例**：

```typescript
// src/components/YCloudMessageSender.test.tsx

import { render, screen, fireEvent, waitFor } from '@testing-library/react';
import { YCloudMessageSender } from './YCloudMessageSender';

describe('YCloudMessageSender', () => {
  test('应该渲染发送消息表单', () => {
    render(<YCloudMessageSender />);
    
    expect(screen.getByLabelText('收件人')).toBeInTheDocument();
    expect(screen.getByLabelText('消息内容')).toBeInTheDocument();
    expect(screen.getByRole('button', { name: '发送' })).toBeInTheDocument();
  });
  
  test('应该验证手机号格式', async () => {
    render(<YCloudMessageSender />);
    
    const phoneInput = screen.getByLabelText('收件人');
    const submitButton = screen.getByRole('button', { name: '发送' });
    
    fireEvent.change(phoneInput, { target: { value: 'invalid' } });
    fireEvent.click(submitButton);
    
    await waitFor(() => {
      expect(screen.getByText('请输入有效的手机号')).toBeInTheDocument();
    });
  });
  
  test('应该成功发送消息', async () => {
    const mockSend = jest.fn().mockResolvedValue({ success: true });
    render(<YCloudMessageSender onSend={mockSend} />);
    
    fireEvent.change(screen.getByLabelText('收件人'), { 
      target: { value: '+1234567890' } 
    });
    fireEvent.change(screen.getByLabelText('消息内容'), { 
      target: { value: 'Hello' } 
    });
    fireEvent.click(screen.getByRole('button', { name: '发送' }));
    
    await waitFor(() => {
      expect(mockSend).toHaveBeenCalledWith({
        to: '+1234567890',
        content: 'Hello'
      });
    });
  });
});
```

### 第三阶段：启动项目并测试（执法堂长老）

**第一步：检查项目环境**

```bash
# 检查后端环境
cd [后端项目路径]
mvn --version
java --version

# 检查前端环境
cd [前端项目路径]
node --version
npm --version
```

**第二步：启动后端服务**

```bash
# 启动后端（后台运行）
cd [后端项目路径]
nohup mvn spring-boot:run > backend.log 2>&1 &

# 等待服务启动
sleep 10

# 检查服务是否启动
curl http://localhost:8080/actuator/health
```

**第三步：启动前端服务**

```bash
# 启动前端（后台运行）
cd [前端项目路径]
nohup npm run dev > frontend.log 2>&1 &

# 等待服务启动
sleep 5

# 检查服务是否启动
curl http://localhost:3000
```

**第四步：执行测试**

```bash
# 执行后端测试
cd [后端项目路径]
mvn test

# 执行前端测试
cd [前端项目路径]
npm test

# 生成覆盖率报告
mvn jacoco:report  # 后端
npm run test:coverage  # 前端
```

**第五步：停止服务**

```bash
# 停止后端
pkill -f "spring-boot:run"

# 停止前端
pkill -f "vite"
```

## 注意事项

1. **测试隔离** - 每次测试前清理测试数据
2. **环境一致** - 确保测试环境与生产环境一致
3. **自动化优先** - 优先编写自动化测试
4. **持续集成** - 将测试集成到 CI/CD 流程
5. **测试数据** - 使用真实但脱敏的测试数据

## TDD 最佳实践

1. **红-绿-重构**
   - 红：先写失败的测试
   - 绿：写最少代码让测试通过
   - 重构：优化代码质量

2. **测试金字塔**
   - 单元测试（70%）- 快速、隔离
   - 集成测试（20%）- 验证模块协作
   - 端到端测试（10%）- 验证完整流程

3. **测试命名规范**
   - `test[功能]_[场景]_[预期结果]`
   - 例如：`testSendMessage_InvalidPhone_ReturnsBadRequest`

4. **断言清晰**
   - 每个测试只验证一个行为
   - 使用描述性的断言消息
