# 天玄宗多 Agent 协作验证清单

## 用户验证清单

当你运行 `/宗门新任务` 后，请检查以下内容来确认是否真正进入了多 Agent 协作模式：

### ✅ 第一步：检查任务目录是否创建

```bash
ls -la 天玄宗/宗门任务榜/
```

**期望结果**：
- 应该看到一个新的任务目录，格式为 `YYYY-MM-DD-任务名称`
- 例如：`2026-03-29-ycloud-auth/`

**如果没有**：
- ❌ 宗主跳过了 Phase 3，没有创建任务
- ❌ 宗主可能自己开始执行任务了

### ✅ 第二步：检查任务文件是否完整

```bash
ls -la 天玄宗/宗门任务榜/2026-03-29-ycloud-auth/
```

**期望结果**：
```
任务总览.md
执行计划.md
阵堂任务.md
器堂任务.md（如有前端）
执法堂验收.md
.task-status.json
```

**如果缺少文件**：
- ❌ 宗主没有完整创建任务文件
- ❌ 可能跳过了 Phase 2 或 Phase 3

### ✅ 第三步：检查是否真正调度了其他 Agent

在 Claude Code 的对话中查看：

**期望看到**：
```
宗主: 🎯 第一步：调度 @阵堂长老
      
      执行模式：单个 Agent

@阵堂长老

**任务文件**: /天玄宗/宗门任务榜/2026-03-29-ycloud-auth/阵堂任务.md

请按照任务文件实现...

---

阵堂长老: 收到！我是阵堂长老，现在开始后端开发工作。
```

**关键标志**：
- ✅ 必须看到 `@阵堂长老` 这样的调用（带 @）
- ✅ 必须看到阵堂长老的回复："收到！我是阵堂长老..."
- ✅ 必须看到工具调用显示从 "宗主" 切换到 "阵堂长老"

**如果只看到**：
```
宗主: 现在开始实现...

⏺ Explore(读取代码)
  ⎿  Done...
  
⏺ 现在开始实现...
```

**这说明**：
- ❌ 宗主没有真正调度其他 Agent
- ❌ 宗主自己在执行任务
- ❌ 失去了多 Agent 协作优势

### ✅ 第四步：检查 Claude Code UI 的 Agent 切换

在 Claude Code 的工具调用面板中：

**期望看到**：
```
⏺ 宗主已就位
  ...需求探索...

⏺ 宗主: 🎯 调度 @阵堂长老

⏺ 阵堂长老: 收到！我是阵堂长老...
  ⎿ Read(...)
  ⎿ Edit(...)
```

**如果一直显示**：
```
⏺ Explore(...)
⏺ Read(...)
⏺ Edit(...)
```

**没有显示 Agent 名称**：
- ❌ 可能没有真正切换 Agent
- ❌ 宗主可能自己在执行

### ✅ 第五步：检查 Git 分支

```bash
git branch
```

**期望结果**：
- 应该切换到新分支：`task/2026-03-29-ycloud-auth`

**如果还在 main 分支**：
- ❌ 宗主没有创建任务分支
- ❌ 跳过了 Phase 3

## 常见问题排查

### 问题 1：宗主说要调度，但没有真正调用

**症状**：
```
宗主: 我会调度阵堂长老来实现...
      
⏺ Explore(读取代码)
  现在开始实现...
```

**原因**：
- 宗主只是说要调度，但没有使用 `@阵堂长老` 格式

**解决**：
- 手动提醒宗主："请使用 @阵堂长老 来调度"
- 或重新运行 `/宗门新任务`

### 问题 2：没有创建任务目录

**症状**：
- `天玄宗/宗门任务榜/` 下没有新目录

**原因**：
- 宗主跳过了 Phase 3

**解决**：
- 停止当前任务
- 提醒宗主："你必须先创建任务目录和文件"
- 或重新运行 `/宗门新任务`

### 问题 3：宗主自己开始写代码

**症状**：
```
⏺ Edit(修改 YCloudClient.java)
⏺ Write(创建 YCloudController.java)
```

**原因**：
- 宗主违反了角色定位，自己执行开发任务

**解决**：
- 立即停止
- 提醒宗主："你是调度者，不是开发者，请调度 @阵堂长老"
- 删除宗主创建的代码
- 重新运行 `/宗门新任务`

## 总结

要确认多 Agent 协作是否生效，检查：

1. ✅ 任务目录已创建
2. ✅ 任务文件完整
3. ✅ 看到 `@阵堂长老` 等调用
4. ✅ 看到长老的开场白："收到！我是xxx长老..."
5. ✅ 工具调用显示 Agent 切换
6. ✅ Git 分支已切换

如果以上任何一项不满足，说明宗主没有遵循多 Agent 协作流程！
