# 宗门任务继续

## 概述

`/宗门任务继续` 命令用于继续执行之前未完成的任务。宗主会检查任务状态，恢复上下文，并继续调度各堂完成剩余工作。

## 执行流程

### 第一阶段：任务恢复（宗主）

0. **🧠 优先：用天玄记忆诀唤醒历史上下文**

   任务恢复前，先检索宗门记忆库，快速回忆此任务的历史轨迹：

   ```bash
   # 搜索与该任务相关的道行观照
   grep -h -i "$TASK_ID\|[任务关键词]" 天玄宗/道行观照/分卷/*.md | head -20

   # 检索对应的宗门记载（如有中途记载）
   grep -i "$TASK_ID\|[任务关键词]" 天玄宗/宗门记载/SUMMARIES.md | head -10
   ```

   如果找到相关观照，优先读取：
   - ✅ 了解上次各堂做了什么、遇到了什么问题
   - ✅ 恢复踩坑记录，避免重蹈覆辙
   - ✅ 快速理解当前代码状态，无需重新探索

1. **验证任务存在**
   ```bash
   if [ ! -d "天玄宗/宗门任务榜/$TASK_ID" ]; then
     echo "❌ 任务不存在: $TASK_ID"
     exit 1
   fi
   ```

2. **读取任务状态**
   - 读取 `.task-status.json`
   - 检查各堂完成情况
   - 识别当前阻塞点

3. **恢复 Git 分支**
   ```bash
   git checkout task/$TASK_ID
   ```

4. **加载任务上下文**
   - 读取任务总览和执行计划
   - 查看已完成的工作
   - 了解当前进度

### 第二阶段：状态分析（宗主）

宗主会分析当前状态并决定下一步：

1. **检查依赖关系**
   - 哪些任务已完成
   - 哪些任务可以开始
   - 哪些任务被阻塞

2. **识别问题**
   - 是否有失败的任务
   - 是否需要调整计划
   - 是否需要用户输入

3. **更新计划**（如需要）
   - 根据实际情况调整
   - 重新分配任务

### 第三阶段：继续执行（宗主）

宗主会继续调度各堂执行：

1. **调度可执行的任务**
   - 检查依赖关系，识别所有可启动的任务
   - 多个任务可同时启动时，默认并行调度
   - 仅有明确阻塞依赖时才串行

2. **监控执行进度**
   - 更新任务状态
   - 处理执行中的问题
   - 并行任务中某一方完成时，等待另一方完成后再继续下一阶段

3. **协调各堂协作**
   - 确保信息同步
   - 解决冲突和问题

## 任务状态说明

### 任务整体状态

- `in_progress` - 进行中
- `blocked` - 被阻塞
- `failed` - 失败
- `completed` - 已完成

### 各堂任务状态

- `pending` - 等待中
- `in_progress` - 进行中
- `completed` - 已完成
- `failed` - 失败

## 配置选项

```bash
# 强制重新开始（忽略已完成的部分）
/宗门任务继续 <任务ID> --restart

# 从指定步骤开始
/宗门任务继续 <任务ID> --from 器堂

# 跳过失败的步骤
/宗门任务继续 <任务ID> --skip-failed
```

## 注意事项

1. **上下文恢复**
   - 宗主会完整恢复任务上下文
   - 确保各堂了解之前的工作

2. **状态同步**
   - 自动切换到任务分支
   - 加载最新的代码变更

3. **问题处理**
   - 失败的任务会重新分析
   - 必要时调整执行计划

4. **用户参与**
   - 重大决策需要用户确认
   - 阻塞问题需要用户输入

## 相关命令

```bash
# 查看所有任务状态
/任务查询

# 创建新任务
/宗门新任务

# 归档完成的任务
/任务归档 <任务ID>
```

## 实现原理

1. **宗主** 读取任务状态文件
2. **宗主** 分析依赖关系和完成情况
3. **宗主** 识别所有可执行的任务（可能有多个可并行）
4. **宗主** 同时调度所有可并行的长老
5. **宗主** 更新任务状态
6. **宗主** 监控执行进度，并行任务全部完成后继续下一阶段

**提示**: `/宗门任务继续` 让你可以随时中断和恢复任务，宗主会智能地恢复上下文并继续执行！
