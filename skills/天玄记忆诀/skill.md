# 天玄记忆诀 (跨会话记忆检索)

跨会话检索**道行观照**与**宗门记载**的三层工作流。先索引、再时间线、最后获取详情，节省约 10x token。

## 何时使用

| 触发场景 | 示例问题 |
|---------|---------|
| 开始新任务 | "这个功能之前实现过吗？" |
| 任务继续 | "上次这个任务走到哪里了？" |
| 遇到 Bug | "这个错误以前解决过吗？" |
| 技术选型 | "之前有没有调研过这个技术？" |
| 理解代码 | "这个模块是谁做的、为什么这么设计？" |

## 三层工作流（节省 10x token）

### 第一层：索引查询（~50 tokens/条）

```bash
# ── 道行观照 ─────────────────────────────────────────────
# 全关键词搜索（最常用）
grep -i "JWT\|认证\|token" 天玄宗/道行观照/OBSERVATIONS.md | head -20

# 按长老过滤（知道是谁做的）
grep "阵堂长老" 天玄宗/道行观照/OBSERVATIONS.md | head -20

# 按类型过滤（知道是什么操作）
grep "· 缺陷修复\|· 功能实现" 天玄宗/道行观照/OBSERVATIONS.md | head -20

# 按日期过滤（知道大概时间）
grep "2026-03" 天玄宗/道行观照/OBSERVATIONS.md | head -20

# ── 宗门记载 ─────────────────────────────────────────────
# 搜索任务关键词
grep -i "用户\|登录\|支付" 天玄宗/宗门记载/SUMMARIES.md | head -15

# 查看最近 3 条记载摘要
grep "^## 📜\|^**任务需求**\|^**学到知识**" 天玄宗/宗门记载/SUMMARIES.md | head -20
```

**目标**：找到相关观照的 **编号**（如 `#023`）。

### 第二层：时间线查询（~200 tokens）

找到编号后，快速了解上下文背景：

```bash
# 查看某条观照的标题和副标题（快速确认是否相关）
grep -A3 "^## #023" 天玄宗/道行观照/OBSERVATIONS.md

# 查看该观照前后的 3 条（了解工作序列）
grep -n "^## #" 天玄宗/道行观照/OBSERVATIONS.md | grep -E "#02[1-5] "

# 查看某天的所有观照（了解当天工作全貌）
grep "^## #.*2026-03-31" 天玄宗/道行观照/OBSERVATIONS.md
```

### 第三层：详情获取（~500-1000 tokens/条）

**只针对筛选后的 1-3 条**获取完整内容：

```bash
# 获取单条观照完整内容（推荐：利用 awk 精确截取）
awk '/^## #023 /,/^## #024 /' 天玄宗/道行观照/OBSERVATIONS.md

# 获取最后一条宗门记载完整内容
awk '/^## 📜 记载/{block=$0; next} block{block=block"\n"$0} END{print block}' \
  天玄宗/宗门记载/SUMMARIES.md

# 获取最近 3 条宗门记载（任务重启时使用）
awk '
  /^## 📜 记载/{if(block!=""){n++;blocks[n]=block}; block=$0; next}
  {if(block!="") block=block"\n"$0}
  END{if(block!=""){n++;blocks[n]=block}; start=(n>3)?n-2:1;
      for(i=start;i<=n;i++) print blocks[i]"\n---"}
' 天玄宗/宗门记载/SUMMARIES.md
```

## 实战示例

### 场景一：新任务开始前查历史

```bash
# 1. 搜索是否有 "支付" 相关历史
grep -i "支付\|payment\|pay" 天玄宗/道行观照/OBSERVATIONS.md | head -15
# 发现 #012 #018 #034 相关

# 2. 快速看标题确认相关性
grep -A2 "^## #012\|^## #018\|^## #034" 天玄宗/道行观照/OBSERVATIONS.md

# 3. #034 最相关（踩坑记录），读取完整内容
awk '/^## #034 /,/^## #035 /' 天玄宗/道行观照/OBSERVATIONS.md
# → 发现：支付回调签名验证有坑，正确方案是...
```

### 场景二：继续中断的任务

```bash
# 1. 搜索任务 ID 相关观照
grep "2026-03-29-user-login" 天玄宗/道行观照/OBSERVATIONS.md | head -10

# 2. 读取最近宗门记载（如有阶段性总结）
tail -n 50 天玄宗/宗门记载/SUMMARIES.md

# 3. 快速了解"上次做到哪了"
grep -A5 "^## #" 天玄宗/道行观照/OBSERVATIONS.md | tail -n 30
```

### 场景三：遇到 Bug 查历史解法

```bash
# 1. 搜索同类错误
grep -i "NPE\|NullPointer\|空指针" 天玄宗/道行观照/OBSERVATIONS.md | head -10

# 2. 过滤缺陷修复类型
grep "· 缺陷修复" 天玄宗/道行观照/OBSERVATIONS.md | grep -i "NPE" | head -5

# 3. 读取解决方案详情
awk '/^## #[编号] /,/^## #[下一编号] /' 天玄宗/道行观照/OBSERVATIONS.md
```

## 各堂典型查询模式

```bash
# 阵堂/器堂 - 开始实现前查同模块历史
grep "阵堂长老\|器堂长老" 天玄宗/道行观照/OBSERVATIONS.md | grep -i "[模块名]"

# 执法堂 - 查同类 Bug 历史解法
grep "· 缺陷修复" 天玄宗/道行观照/OBSERVATIONS.md | tail -20

# 藏经阁 - 查是否已有调研
grep "· 典籍调研" 天玄宗/道行观照/OBSERVATIONS.md | grep -i "[技术名]"

# 宗主 - 查任务全局历史
grep "· 宗门决策" 天玄宗/道行观照/OBSERVATIONS.md | head -20
grep "^**下一步骤**" 天玄宗/宗门记载/SUMMARIES.md | head -10
```

## Token 对比

| 方式 | Token 消耗 | 建议 |
|------|-----------|------|
| 第一层索引 | ~50/条 | 先用这个确认存在 |
| 第二层时间线 | ~200 | 了解背景上下文 |
| 第三层详情 | ~500-1000/条 | 只读 1-3 条最相关的 |
| ❌ 直接全读文件 | ~5000-50000 | 绝对不要这样做 |
